# Переменные и типы данных
  

Переменные невероятно важны, так как позволяют хранить информацию и использовать её в дальнейшем . Вначале может быть не совсем понятно зачем вообще что-то записывать в переменную, если можно просто оперировать значениями без них. Понимание переменных придет немного позже, когда мы начнем создавать более сложные программы и нам потребуется хранить информацию в каком-либо месте. Переменные используются для хранения значений. Простыми словами Переменная – это именованная область памяти, имеющая определенный тип. Данные, находящиеся в переменной (то есть по ее адресу в памяти ), называются значением этой переменной.  

Названия переменной может включать в себя символы английского алфавита, цифры и знак подчёркивания. Переменная не должна совпадать с ключевыми словами (это специальные слова, которые используются в качестве управляющих конструкций , для определения типов и т. п. )   

Все переменные в языке С должны быть объявлены перед использованием. При использовании в программе ранее не объявленной переменной компилятор выдаст сообщение об ошибке вроде..
 
**test.c: In** **function 'main':**

**test.c:7: error: 'x' undeclared (first use in this function)**

**test.c:7: error: (Each undeclared identifier is reported only once**

**test.c:7: error: for each function it appears in.)**

  

В  этом сообщении указывается имя файла и номер строки , в которой обнаружена ошибка (test.c:5), а также описание ошибки ('x' undeclared -- объект с именем x не объявлен).
  

При объявлении переменной мы должны указать ее тип . Существует несколько стандартных типов, но программист может создавать и собственные типы данных.

  

## Типы данных


Типы переменных в языке Си указываются перед названием переменной. От конкретного типа данных будет зависеть содержимое переменной. Если мы укажем что переменная с типом данных для целых чисел int, то в неё мы не сможем записать строку или число с точкой.


**Тип переменной определяет:**
  

1.  Размер переменной в байтах (сколько байт памяти выделит компьютер для хранения значения)
      

2.  Представление переменной в памяти (как в двоичном виде будут расположены биты в выделенной области памяти).
     

В  Си несколько основных типов. Разделим их на две группы - целые и числа с плавающей точкой.
    



## Целые
 

- char - размер 1 байт. Всегда! Это нужно запомнить.

- short - размер 2 байта

- int - размер 4 байта

- long - размер 4 байта

- long long - размер 8 байт.


Здесь следует сделать замечание. Размер переменных в си не определён явно, как размер в байтах. В стандарте только указано, что

  

char <= short <= int <= long <= long long

  

Указанные выше значения характерны для компилятора VC2012 на 32-разрядной машине. Так что, если ваша программа зависит от размера переменной, не поленитесь узнать её размер.

    

Теперь давайте определим максимальное и минимальное число, которое может хранить переменная каждого из типов.

Числа могут быть как положительными, так и отрицательными. Отрицательные числа используют один бит для хранения знака. Иногда знак необходим (например, храним счёт в банке, температуру, координату и т.д.), а иногда в нём нет необходимости (вес, размер массива, возраст человека и т.д.). Для этого в си используется модификатор типа signed и unsigned. unsigned char - все 8 бит под число, итого имеем набор чисел от 00000000 до 11111111 в  двоичном виде, то есть от 0 до 255 signed char от -128 до 128. В си переменные по умолчанию со знаком. Поэтому запись char и signed char эквивалентны.

short отличается от int в том что в нем можно записать только малое количество число ( с небольшим размером ), а если запишите большое количество чисел ( число большого размера ), у вас будет ошибка. Простыми словами, short нужен для чисел небольшого размера.

  

  

unsigned – говорить о том что данные в памяти могут быть только положительными, то есть мы не можем записать в этот тип отрицательные чила, только положительные.

signed - говорить о том что данные в памяти могут быть и отрицательным и положительным, то есть мы можем записать в этот тип и отрицательные чила и положительные.

Если вы хотите знать в переменной число с плавающей точкой, например : 2.43, 7.64, 53.2 и т. д, используйте тип float, но если вы хотите хранить в переменной число с плавающей точкой большого размера , например: 4.9267282626, 43.326272728 и т. д, то используйте тип double, это означает что вы работаете с более точными цифрами. Когда хотите хранить символические данные, например: латинские буквы, разные символы и тому подобное , используйте переменную типа char. Тип данных char предназначен для хранения одного символа и занимает 1 байт памяти. Каждому символу соответствует 8-битное целое значение – ASCII-код. Таким образом тип char может использоваться для хранения целых чисел от -128 до 127 или от 0 до 255 (в зависимости от того, какой из типов используется : signed char и unsigned char).  

'0'..'9' – цифры в формате символов (ASCII-коды: 48-57)
  

'A'..'Z' – заглавные (прописные) латинские буквы (ASCII-коды: 65-90)
  

'a'..'z' – маленькие ( строчные) латинские буквы (ASCII-коды: 97- 122)


Символьные переменные сравнимы согласно значениями их ASCII-кодов. При использовании совместно с целыми типами возможно приведение от одного типа к другому согласно ASCII.

### Типы данных в языке Си

В языке Си существуют следующие основные типы данных:

1. **Целочисленные типы (integer types)**:
    
    - `char`: используется для хранения символов, но также может использоваться для целых чисел. Обычно занимает 1 байт (8 бит).
    - `int`: используется для хранения целых чисел. Размер может варьироваться, но часто составляет 4 байта.
    - `short int` (`short`): обычно занимает 2 байта.
    - `long int` (`long`): обычно занимает 4 или 8 байт в зависимости от системы.
    - `long long int` (`long long`): предназначен для хранения очень больших целых чисел, обычно занимает 8 байт.
2. **Типы с плавающей точкой (floating-point types)**:
    
    - `float`: используется для хранения чисел с плавающей запятой одинарной точности. Обычно занимает 4 байта.
    - `double`: для хранения чисел с плавающей запятой двойной точности. Обычно занимает 8 байт.
    - `long double`: для хранения чисел с плавающей запятой повышенной точности. Размер может варьироваться, но часто составляет 10, 12 или 16 байт.
3. **Типы данных для работы с указателями (pointer types)**:
    
    - `void*`: универсальный указатель, который может указывать на любой тип данных.
    - Указатели на другие типы, например, `int*`, `char*`.
4. **Типы данных для работы с массивами (array types)**:
    
    - Массивы фиксированной длины, например, `int arr[10];`.
5. **Структуры и объединения (structs and unions)**:
    
    - `struct`: для объединения разных типов данных в одну сущность.
    - `union`: позволяет хранить несколько типов данных, но только один из них используется в данный момент времени.
6. **Пользовательские типы данных (user-defined types)**:
    
    - `typedef`: для создания новых имен типов на основе уже существующих.

### Зачем нужны типы данных

1. **Управление памятью**: Каждый тип данных занимает определенное количество памяти. Зная тип данных, компилятор может выделить необходимое количество памяти для хранения значения.
    
2. **Семантическое значение**: Тип данных определяет, какие операции могут быть выполнены с данным значением. Например, над целыми числами можно выполнять арифметические операции, а над указателями - операции разыменования и сдвига.
    
3. **Защита от ошибок**: Типы данных помогают избежать ошибок, связанных с неправильным использованием данных. Например, попытка сложить строку с числом приведет к ошибке компиляции.
    
4. **Оптимизация производительности**: Знание типов данных позволяет компилятору оптимизировать код для более эффективного выполнения. Например, операции с целыми числами обычно выполняются быстрее, чем с числами с плавающей запятой.
    

### Типизация данных

Типизация данных – это концепция, описывающая, как язык программирования управляет типами данных и их проверкой.

1. **Статическая типизация**: Типы данных проверяются на этапе компиляции. Это позволяет обнаружить ошибки до выполнения программы. Примеры: Си, C++, Java.
    
2. **Динамическая типизация**: Типы данных проверяются во время выполнения программы. Это позволяет писать более гибкий код, но повышает риск ошибок в рантайме. Примеры: Python, JavaScript.
    
3. **Сильная типизация**: Язык строго ограничивает преобразования типов и не позволяет смешивать значения разных типов без явного преобразования. Примеры: Haskell, Python.
    
4. **Слабая типизация**: Язык допускает автоматическое преобразование типов, что может привести к неожиданным результатам. Примеры: JavaScript, PHP.
    

Типы данных и типизация помогают разработчикам создавать более надежные, эффективные и предсказуемые программы. В языке Си они играют ключевую роль в управлении памятью и обеспечении производительности приложений.
